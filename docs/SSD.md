# System Specification Document (SSD)
**Проєкт:** PHP News CMS & Blog
**Автор:** Софія Котюк, ІПЗ-23-1


## 1. Мета та обсяг системи
**Мета:** розробити повноцінну систему управління контентом (CMS) для публікації, редагування та керування статтями, а також для взаємодії з читачами через систему коментарів.
**Обсяг:**
1. **Публічний інтерфейс(Frontend):** Доступний для всіх відвідувачів. Включає стрічку новин з пагінацією, фільтрацію за категоріями, пошук, перегляд окремих публікацій, асинхронне додавання коментарів та перегляд профілів авторів.
2. **Адміністративна панель(Backend):** Захищена зона для керування контентом. Включає дашборд зі статистикою, управління публікаціями (CRUD), категоріями, модерацію коментарів, керування профілями адміністраторів та налаштування особистого профілю.

## 2. Функціональні вимоги
### Сценарії використання для Читача 
* **Перегляд стрічки:** Користувач може переглядати список публікацій, які виводяться по 5 на сторінку (пагінація).
* **Пошук та фільтрація:** Користувач може шукати статті за ключовими словами (пошук відбувається по заголовку, категорії або тексту) та фільтрувати їх за обраною категорією.
* **Перегляд публікації:** Користувач може відкрити конкретну статтю для детального читання, перегляду зображення та коментарів до неї.
* **Коментування:** Користувач може залишити коментар (ім'я, email, текст) без перезавантаження сторінки (AJAX). Для захисту від спаму діє обмеження: один коментар не частіше ніж раз на 120 секунд.
* **Профілі авторів:** Користувач може переглядати публічну сторінку автора (адміністратора) з його фотографією, заголовком та біографією.

### Сценарії використання для Адміністратора
* **Автентифікація:** Адміністратор може увійти в систему за допомогою унікального `username` та `password`. Після входу сесія зберігається.
* **Дашборд:** На головній панелі адміністратор бачить загальну статистику: кількість постів, категорій, адміністраторів, коментарів, а також список топ-6 останніх постів.
* **Управління статтями (Posts):** Адміністратор може створювати нові публікації, редагувати існуючі та видаляти їх. При створенні можна завантажити зображення (JPG, JPEG, PNG, GIF). При видаленні поста асоційоване зображення також видаляється з сервера.
* **Управління категоріями:** Адміністратор може додавати нові категорії (назва від 3 до 49 символів) та видаляти їх. Система не дозволяє створювати дублікати категорій.
* **Модерація коментарів:** Адміністратор бачить список усіх коментарів і може видаляти їх за необхідності.
* **Управління адміністраторами:** Адміністратор може створювати нових користувачів із доступом до панелі керування (пароль має містити більше 3 символів) та видаляти існуючих.
* **Особистий профіль:** Адміністратор може оновлювати власне ім'я, заголовок (до 30 символів), біографія (до 500 символів) та завантажувати нове фото профілю (аватар).

## 3. Нефункціональні вимоги
### Безпека
* **Захист від SQL-ін'єкцій:** Всі запити до бази даних виконуються виключно через підготовлені вирази (Prepared Statements) з використанням бібліотеки PDO.
* **Контроль доступу:** Кожен контролер адміністративної панелі викликає метод `requireLogin()`, який перевіряє наявність `UserId` у сесії. Неавторизовані користувачі перенаправляються на сторінку входу.
* **Валідація файлів:** Завантаження файлів (зображення постів, аватари) суворо перевіряється за розширенням.
* **Захист від спаму:** Реалізовано сесійний контроль частоти запитів для додавання коментарів (`commentAsync`).

### Продуктивність та Архітектура
* **Шаблон проєктування:** Використовується класичний патерн MVC. Маршрутизація здійснюється через єдину точку входу (`index.php`) за параметрами `controller` та `action`.
* **Обробка помилок:** Реалізовані кастомні сторінки для помилок 404 (Not Found) та 500 (Internal Server Error).
* **Повідомлення користувачам:** Використовується механізм Flash-повідомлень через сесії (`setErrorMessage`, `setSuccessMessage`) для інформування про результати дій.

## 4. Інтерфейси та Формати даних
* **REST/AJAX API:** Система надає ендпоінт `commentAsync` (метод POST), який приймає та повертає дані у форматі JSON для динамічного оновлення коментарів.
* **Користувацький інтерфейс (UI):** Frontend побудований з використанням фреймворку Bootstrap 4.2.1, що забезпечує адаптивність для мобільних пристроїв та десктопів. Використовуються іконки FontAwesome.

## 5. Обмеження та Стек технологій
* **Мова програмування:** Нативний PHP.
* **База даних:** MySQL (база даних `news_db`).
* **Вебсервер:** Будь-який сервер із підтримкою PHP (Apache, Nginx, локальний XAMPP/MAMP).
* **Файлова система:** Зображення зберігаються локально у директорії `public/uploads/`.
